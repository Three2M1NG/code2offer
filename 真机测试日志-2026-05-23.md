# code2offer 安卓真机测试日志

日期: 2026-05-22 ~ 2026-05-23
设备: Redmi 22021211RC (Android 14 / MIUI)

---

## 1. APK 编译失败 — record 包版本冲突

**根因**: `record 5.2.0` 的子包 `record_linux 0.7.2` 与 `record_platform_interface 1.5.0` 不兼容。`record_linux` 缺少 `startStream` 方法实现，且 `hasPermission` 方法签名不匹配。

**解决方案**: 升级 `record` 到 `^6.2.0`，删除 `dependency_overrides`。

**如何修复**: `pubspec.yaml` 中 `record: ^5.2.0` → `record: ^6.2.0`，运行 `flutter pub get`。

---

## 2. "Failed to load problems"（第一次）

**根因**: 数据库 `problems` 表（旧版 schema 遗留）与 Python ORM 模型不匹配：
- 表缺少 `title_cn`、`created_at`、`updated_at` 列
- `tags` 字段类型冲突：ORM 定义为 `relationship(ProblemTag)`，数据库实际是 `TEXT[]` 数组列
- 查询时 SQLAlchemy 抛 `UndefinedColumn` 异常

**解决方案**:
1. `ALTER TABLE problems ADD COLUMN` 补建缺失列
2. 修改 ORM 模型：`tags = mapped_column(ARRAY(String))`，删除不存在的 `Tag`/`ProblemTag` 关系类

**如何修复**: Docker exec psql 执行 DDL，重写 `models.py` 和 `routers/problems.py`。

---

## 3. 语音识别报错 — ASR API Key 未加载

**根因**: `.env` 文件中变量名 (`DASHSCOPE_API_KEY`、`ZHIPU_API_KEY`) 与 `config.py` 期望的 (`BAILIAN_API_KEY`) 不一致，ASR 客户端 `configured` 为 False。

**解决方案**: 更新 `config.py`，使用 fallback 逻辑同时支持两套变量名：
```python
BAILIAN_ASR_API_KEY = os.getenv("DASHSCOPE_API_KEY", "") or os.getenv("BAILIAN_ASR_API_KEY", "")
BAILIAN_API_KEY = os.getenv("BAILIAN_API_KEY", "") or os.getenv("ZHIPU_API_KEY", "")
```

**如何修复**: 修改 `config.py` Settings 类，`asr_client.py` 改用独立的 `BAILIAN_ASR_API_KEY`。

---

## 4. 语音识别报错 — ASR API 端点错误

**根因**: `qwen3-asr-flash` 模型使用 **OpenAI 兼容接口**（`/compatible-mode/v1/chat/completions`，`input_audio` 格式），不是 DashScope 原生 ASR 端点 (`/services/audio/asr/transcription`)。原代码请求格式完全不对。

**解决方案**: 重写 `QwenASRClient`：
- 端点改为 `https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions`
- 请求体改为 chat completion 格式：`messages[0].content[0] = {"type": "input_audio", "input_audio": {"data": "data:audio/mp4;base64,..."}}`
- 响应解析改为 `choices[0].message.content`

**如何修复**: 完全重写 `asr_client.py` 的 `QwenASRClient` 类。

---

## 5. "Failed to load problems"（第二次 — 代码被还原）

**根因**: `models.py` 和 `problems.py` 被 linter/用户还原为 M:N 标签架构（`Tag`/`ProblemTag` 关系表 + `tag_links` relationship）。但数据库恰好也已被迁移到 M:N 架构（`tags` 和 `problem_tags` 表已存在且有数据），代码与数据库再次一致。

**解决方案**: 无需修改代码。数据库和代码恰好匹配。

**如何修复**: 识别到 schema 已同步，保持现有代码。

---

## 6. "题目检索失败，请重试"

**根因**: `retriever.py` 使用 `psycopg.connect(settings.DATABASE_URL)` 直连数据库，但 `.env` 中 `DATABASE_URL=postgresql+psycopg://...`。`+psycopg` 是 SQLAlchemy 方言前缀，psycopg 驱动无法解析，连接失败。

**解决方案**: 在 `_get_conn()` 中增加 URL 规范化，自动去除 `+psycopg` 方言指定：
```python
if db_url.startswith("postgresql+"):
    db_url = db_url.replace("postgresql+psycopg", "postgresql", 1)
```

**如何修复**: 修改 `retriever.py` 的 `_get_conn` 方法。

---

## 7. 所有题目都按"链表反转"评价

**根因**: RAG 检索器只对用户输入文本做向量相似度搜索，完全忽略了用户在 App 中选择的 `problem_id`。无论选哪道题，匹配结果只取决于文本描述和 embedding 相似度。

**解决方案**: 
1. `retriever.py` 新增 `get_by_id()` 方法，直接按 UUID 查题
2. `analyze.py` 的 `_stream_analysis` 增加 `problem_id` 参数，优先走精确查题，其次向量检索
3. Flutter 端 `api_service.dart`、`analysis_provider.dart`、`analysis_page.dart` 全链路传递 `problemId`

**如何修复**: 修改后端 `retriever.py`、`routers/analyze.py`，修改前端 `api_service.dart`、`analysis_provider.dart`、`analysis_page.dart`。

---

## 8. 文字输入发送按钮无效

**根因**: 发送按钮的 `onPressed` 在 build 时计算：`_textController.text.trim().isNotEmpty ? _submit : null`。但 `TextField` 缺少 `onChanged` 回调，用户输入后不触发 `setState`，按钮永远保持 build 时的 disabled 状态。

**解决方案**: 添加 `onChanged: (_) => setState({})`，每次输入触发重建，重新求值 `onPressed`。

**如何修复**: 一行代码：`onChanged: (_) => setState(() {}),`。

---

## 9. 语音转录后不自动评判

**根因**: `_stopRecording()` 只调用了 `api.transcribeAudio()`，转录文本填入输入框后就结束了，没有调用 `analyze()` 触发 AI 评判。加上发送按钮也坏了（问题 8），用户无法提交。

**解决方案**: 转录成功后自动调用 `analyze(text, problemId: widget.problemId)`，并翻转卡片展示评判结果。

**如何修复**: 在 `_stopRecording()` 的转录成功分支末尾增加：
```dart
ref.read(analysisProvider.notifier).reset();
ref.read(analysisProvider.notifier).analyze(text, problemId: widget.problemId);
if (!_isFlipped) _flipCard();
```

## 10. 麦克风按钮 — 长按手势不可靠

**根因**: `GestureDetector` 的 `onLongPressStart`/`onLongPressEnd` 在 Redmi/Android 上不可靠。日志显示录音启动后持续 20+ 秒未停止，说明 `onLongPressEnd` 从未触发。

**解决方案**: 改为点按切换模式——`onTap: isRecording ? onStop : onStart`，点一下开始，再点一下停止。

**如何修复**: 重写 `_MicButton`，从 long-press 改为 tap-toggle。

---

## 11. BoxConstraints 布局溢出

**根因**: 录音时 `_MicButton` 的 `AnimatedContainer` 尺寸随音量动态增长（`56 + amplitude * 20`），超出 `Row` 中可用空间，触发 Flutter 布局约束冲突。

**解决方案**: 用 `SizedBox(width: 56, height: 56)` 固定按钮尺寸，仅保留 glow 效果在 `boxShadow` 中变化（不影响布局）。

**如何修复**: 包裹 `SizedBox`，移除动态 width/height。

---

## 12. dart:ui/painting.dart 渲染错误（黄字报错）

**根因**: `AnimatedContainer` + `BoxShadow` + `Chip(padding: EdgeInsets.zero)` 复杂组件组合在部分 Android 设备上触发 `dart:ui` 底层渲染异常。

**解决方案**:
1. `_MicButton` 从 `AnimatedContainer` + `BoxShadow` 简化为普通 `Container` + 纯色背景
2. 标签展示从 `Chip` 替换为 `Container` + `Text`，避免 Material Chip 的渲染限制

**如何修复**: 移除动画和阴影，使用最简单的容器组件。

## 总结

| # | 问题 | 类别 | 根因 |
|---|------|------|------|
| 1 | 编译失败 | 依赖管理 | record 包版本冲突 |
| 2 | 题目加载失败 | 数据库 | schema 与 ORM 模型不一致 |
| 3 | 语音识别不可用 | 配置 | 环境变量命名不匹配 |
| 4 | 语音识别不可用 | API | 端点/请求格式错误 |
| 5 | 题目加载失败(复发) | 代码同步 | linter 还原代码，DB 也已迁移 |
| 6 | 题目检索失败 | 数据库驱动 | URL 方言前缀不兼容 |
| 7 | 评判题目错误 | 业务逻辑 | RAG 忽略用户选择的题目 |
| 8 | 发送按钮无效 | UI | 缺少 onChanged 触发重建 |
| 9 | 转录后不评判 | 业务逻辑 | 流程中断 |
| 10 | 录音无法停止 | UI 手势 | long-press 手势不可靠 |
| 11 | 布局溢出 | UI 布局 | 动态尺寸超出约束 |
| 12 | 渲染报错 | UI 渲染 | 复杂组件触发底层异常 |

核心教训：
- **数据库**: schema 变更需要迁移脚本，不能让 Docker volume 持久化旧结构而代码已更新
- **API 集成**: 使用第三方 API 前必须先确认端点格式和模型名称（DashScope ASR 有原生和 OpenAI 兼容两套接口）
- **配置管理**: `.env` 变量名需要与代码约定保持一致，或代码做兼容处理
- **Flutter 手势**: `onLongPressEnd` 在部分 Android 设备不可靠，点按切换更稳定
- **Flutter 布局**: `AnimatedContainer` 动态尺寸需注意父级约束，简单组件优先
